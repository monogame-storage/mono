# Mono Android Wrapper

Generic Android wrapper for packaging any Mono game as an APK.

## Mono Engine

Mono is a constraint-driven fantasy game console:
- **Resolution:** 160x144 pixels
- **Colors:** 16 grayscale shades (4-bit mode)
- **Scripting:** Lua 5.4 (via Wasmoon)
- **API:** `cls`, `rect`, `rectf`, `circ`, `circf`, `line`, `pix`, `text`, `spr`, `cam`, `note`, `btn`, `btnp`

Full API reference: see `cart/.mono/CONTEXT.md` (generated by Mono editor)

## Quick Start

1. Create a project from the Mono repo:
   ```bash
   ./scripts/create-android.sh ~/my-game --project-name "My Game" --org com.example
   ```

2. Write your game in `cart/main.lua`:
   ```lua
   local scr = screen()

   function _init()
     mode(4)  -- 16 colors
   end

   function _draw()
     cls(scr, 0)
     text(scr, "HELLO MONO!", 80, 72, 15, ALIGN_HCENTER)
   end
   ```

3. Run on device:
   ```bash
   cd ~/my-game && ./run-debug.sh
   ```

## Game Structure

```
my-game/
  cart/                         # YOUR GAME FILES
    main.lua                    # Entry point (required)
    shader.json                 # Shader config (chain + params)
    .mono/                      # Engine files (auto-generated)
      engine.js                 # Mono engine
      console-gamepad.js        # Virtual touch gamepad
      shader.js                 # Shader pipeline
      shaders/                  # Shader modules (tint, lcd, crt, etc.)
      CONTEXT.md                # API reference & docs (read this!)
      VERSION                   # Engine version
  app/                          # Android app (template)
  run-debug.sh                  # Build + install + launch
  update-icon.sh                # Replace app icon from PNG
```

## Writing Games

Mono games use a simple lifecycle:

```lua
local scr = screen()

function _init()    -- called once: set mode, load assets
  mode(4)           -- 1=2色, 2=4色, 4=16色
end

function _start()   -- called once after _init: init game state
end

function _update()  -- called every frame (30fps): game logic
end

function _draw()    -- called every frame after _update: rendering
  cls(scr, 0)
end
```

### Key APIs

| Function | Description |
|---|---|
| `screen()` | Get main screen surface |
| `cls(scr, color)` | Clear screen |
| `rect(scr, x, y, w, h, color)` | Draw rectangle outline |
| `rectf(scr, x, y, w, h, color)` | Draw filled rectangle |
| `circ(scr, x, y, r, color)` | Draw circle outline |
| `circf(scr, x, y, r, color)` | Draw filled circle |
| `line(scr, x0, y0, x1, y1, color)` | Draw line |
| `pix(scr, x, y, color)` | Set pixel |
| `text(scr, str, x, y, color, align)` | Draw text |
| `btn("up"/"down"/"left"/"right"/"a"/"b"/"start"/"select")` | Button held |
| `btnp(key)` | Button just pressed |
| `note(channel, "C4", duration)` | Play sound (channel 0-1, note "C0"-"B8") |
| `cam(x, y)` | Set camera offset |
| `SCREEN_W`, `SCREEN_H` | Screen dimensions (160, 144) |
| `ALIGN_HCENTER`, `ALIGN_RIGHT`, `ALIGN_CENTER` | Text alignment |

**Colors:** 0 (black) to 15 (white) in 16-color mode.

For the complete API, open `cart/.mono/CONTEXT.md` after creating a project.

## Shader Config

`cart/shader.json` controls post-processing effects:

```json
{
  "chain": ["tint", "lcd"],
  "params": {
    "tint": { "tint": [1.0, 0.75, 0.3] },
    "lcd": { "thickness": 0.20, "pixel_size": 1.0, ... },
    "crt": { "curvature": 0.02, "vignette": 0.1 },
    "scanlines": { "opacity": 0.2 }
  }
}
```

Edit `chain` array to enable/disable shaders. All params have defaults.

## Build Commands

```bash
./run-debug.sh              # Build + install + launch (incremental)
./run-debug.sh --clean      # Clean build
./update-icon.sh icon.png   # Replace app icon (512x512 PNG recommended)
./gradlew assembleDebug     # Build debug APK only
./gradlew bundleRelease     # Build release AAB (requires signing setup)
```
